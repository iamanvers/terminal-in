# Training the 3B model on a Colab GPU

The real 3B base (`Qwen2.5-3B-Instruct`) does **not** train on the 16 GB CPU
laptop — `bf16` is software-emulated on Zen3 (~90-day ETA) and `fp32` weights
(12.4 GB) swap to disk. A free Colab **T4 has 16 GB of real VRAM + native fp16**,
so QLoRA fine-tunes it in ~1–2 h. (See `memory/hardware_training_ceiling.md`.)

The dataset blends public corpora with the system's **own** closed trades +
hindsight-judged decisions from the local DB — which Colab can't see. So we build
the dataset here and carry it up.

## Round trip

**1 · Build the dataset locally** (bundles your own trades + decisions):
```powershell
.venv\Scripts\python.exe scripts\export_dataset.py
# → data\training\colab\dataset.jsonl   (currently ~5,663 samples, ~5 MB)
```

**2 · Train on Colab:**
- Open <https://colab.research.google.com>, upload `colab/train_3b_colab.ipynb`.
- **Runtime → Change runtime type → T4 GPU.**
- Run the cells: install → upload `dataset.jsonl` → train (QLoRA, recipe-matched
  to `train_lora.py` so it's apples-to-apples with the eval gate) → download.
- Cell 5 downloads the **LoRA adapter** (~50–130 MB). Cell 6 (optional) produces
  a ready **q8_0 GGUF** instead.

**3 · Deploy locally + eval-gate:**

*Adapter path (recommended — reuses the proven deploy pipeline):*
```
# unzip the adapter into the nested layout the deployer expects:
#   data/training/runs/<run_id>/adapter/adapter/
$env:LORA_BASE_MODEL = 'Qwen/Qwen2.5-3B-Instruct'
# then merge → GGUF Q4_K_M → ollama create financial-analyst-vN:
#   POST /api/training/deploy  {"run_id": "<run_id>"}
```

*GGUF path:* drop the `.gguf` beside a `Modelfile` (`FROM ./financial-3b-q8_0.gguf`)
and `ollama create financial-analyst-v3 -f Modelfile`.

**4 · Gate before promoting** — the 42-item eval set correctly rejected the 1.1B
v2 (9.5% vs qwen 83.3%); don't repeat that:
```powershell
.venv\Scripts\python.exe -m terminal_in.agents.training.evalset --model financial-analyst-v3
```
Only switch the planner/analyst (settings dropdown) if it **beats the incumbent**
with no >5pt category regression.

## Files
- `scripts/export_dataset.py` — builds the upload-ready `dataset.jsonl`.
- `colab/train_3b_colab.ipynb` — the Colab notebook (generated by `_build_notebook.py`).
- `colab/_build_notebook.py` — edit the notebook **here**, never the `.ipynb` by hand.

## Notes
- The notebook keeps the LoRA config identical to the local recipe (r16/α32,
  q/k/v/o, lr 2e-4, cosine, eff. batch 16, 3 epochs) so the eval comparison is fair.
- A T4 trains all 3 epochs of ~5.7k samples in ~1–2 h; an L4/A100 runtime is faster.
- The personal-data adapter is **non-distributable** (see `docs/LEGAL.md`) — it
  trains on your own trades. Keep it local; only public-corpora models ship.
